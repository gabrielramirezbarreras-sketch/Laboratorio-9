Practica 1: Instalacion y acceso a webmin
Paso 1: Actualizar el sistema.
sudo apt update

Paso 2: Instalar dependencias.
sudo apt install wget apt-transport-https software-properties-common -y

Paso 3: Descargar Webmin.
wget https://www.webmin.com/download/deb/webmin-current.deb

Paso 4: Instalar Webmin.
sudo dpkg -i webmin-current.deb

Si hay errores de dependencias:
sudo apt --fix-broken install -y

Acceso a Webmin:
https://localhost:10000

Credenciales:
Usuario: tu usuario de Linux
Contraseña: tu contraseña

Practica 3: Instalación de Ansible.
Instalar Ansible:
sudo apt update
sudo apt install ansible -y

Verifica:
ansible --version

Instalar dependencias necesarias:
sudo apt install sshpass openssh-client -y

Crear usuario "ansible" (en el controller):
sudo adduser ansible

Agregar a sudoers:
sudo usermod -aG sudo ansible

Cambiar a ese usuario:
su - ansible

Configurar acceso SSH sin contraseña:
ssh-keygen -t rsa -b 2048

Copiar clave:

ssh-copy-id ansible@192.168.140.129

Crear usuario ansible también:
sudo adduser ansible
sudo usermod -aG sudo ansible

Crear Directorio:
sudo mkdir -p /etc/ansible

Editar:
sudo nano /etc/ansible/hosts

Agregar:
[linux]
linux1 ansible_host=192.168.140.129 ansible_user=ansible

[win]
windows1 ansible_host=192.168.140.1 ansible_user=ansible ansible_connection=ssh

Probar conexión con ping:
ansible linux -m ping

Practica 4: Comandos Ad-Hoc.
Crear Direcotrio:
mkdir -p /home/ansible/Desktop

Poner informacion:
echo "Prueba Ansible" > ~/Desktop/archivo.txt

Verifica que existe:
ls ~/Desktop/archivo.txt

Créalo de nuevo correctamente
nano /home/ansible/hosts

PEGA:
[linux]
linux1 ansible_host=192.168.140.129 ansible_user=ansible

Prueba inventario:
ansible-inventory -i /home/ansible/hosts --list

Comando correcto:
ansible linux -i /home/ansible/hosts -m ping -u ansible --ask-pass

Reiniciar la VM con reboot:
ansible linux -i ./hosts -m reboot -b -K
